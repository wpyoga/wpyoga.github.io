Installing Proxmox VE on Debian Bullseye

January 26, 2022

I tried to install Proxmox VE 7.1 as a VM using libvirt, but ran into two issues:

  • The official ISO image cannot be booted as a regular Linux installation ISO, because it does not have Joliet extensions
  • The official installer is GUI-based, so we cannot install it over a serial console

Backgroundâ€‹

I wanted to learn Proxmox. Since I already have a VM server using libvirt, I figured I should install Proxmox VE as a virtual machine. I've installed OpenVZ 6 and 7 before as virtual machines, so it should be straightforward.

However, as mentioned above, I ran into two issues. Therefore I will try to install Proxmox on Debian Bullseye instead, following this tutorial: https://pve.proxmox.com/wiki/Install_Proxmox_VE_on_Debian_11_Bullseye

Planningâ€‹

Base OS: Debian 11.2.0 Network: 192.168.126.0/24

Base OS installationâ€‹

First, install Debian 11 as usual. The thing about Debian installers is that the location we choose determines our timezone. Suppose I live in Asia, and I would like my laptop to have the English (US) locale, what should I do?

Therefore I usually pretend I'm in the USA, then change the timezone after installation is complete. This is much easier than changing the locale.

Proxmox-specific changesâ€‹

After installation, /etc/hosts will look something like this:

127.0.0.1   localhost
127.0.1.1   my-hostname

# The following lines are desirable for IPv6 capable hosts
::1     localhost ip6-localhost ip6-loopback
ff02::1 ip6-allnodes
ff02::2 ip6-allrouters

First, check the ip address of the node. An internal IP address is fine. This can be done using ip addr.

Remove the line 127.0.1.1 my-hostname and replace it with the reachable IP address:

192.168.122.91  my-hostname

Note to self: submit a correction or register an account: https://forum.proxmox.com/threads/how-can-we-contribute-to-the-wiki.93970/

Rebootâ€‹

Now we shut down the VM, and enable nested virtualization for the libvirt domain: https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/8/html/configuring_and_managing_virtualization/creating-nested-virtual-machines_configuring-and-managing-virtualization

Then start the domain again, which will boot into the Proxmox kernel.

Create a non-root userâ€‹

$ sudo pveum useradd william@pve
$ sudo pveum passwd william@pve

Note that pve is the realm. This is not the node's hostname.

Also create a group for the user:

$ sudo pveum group add admin
$ sudo pveum user modify william@pve -append -groups admin

Then grant permissions to the group:

$ sudo pveum acl modify / --roles Administrator --groups admin

Web GUIâ€‹

Open https://192.168.122.91:8006/ in a browser. Note that this is only accessible via HTTPS. Plain HTTP won't work.

Note to self: try to configure a redirect here (HTTP -> HTTPS)

Run Proxmox without a subscriptionâ€‹

Remove /etc/apt/sources.list.d/pve-enterprise.list

Note to self: try to figure out how to remove this automatically.

Create a network bridgeâ€‹

Since my home server is connected to the router via Wi-Fi, let's create a bridge for a routed network. https://192.168.122.91:8006/pve-docs/chapter-sysadmin.html#_routed_configuration

The bridge should get an IP address of .1 in a new subnet, and there should be no interface added to it. In my example, the CIDR is 192.168.126.1/24.

Physical host networkâ€‹

on the physical host

$ sudo ip route add 192.168.126.0/24 via 192.168.122.91

or:

$ virsh net-edit default && virsh net-destroy default && virsh net-start default && sudo systemctl restart libvirtd
<network>
  ...
  <route family='ipv4' address='192.168.126.0' prefix='24' gateway='192.168.122.91'/>
</network>
$ sudo iptables -I FORWARD -s 192.168.126.0/24 -i virbr0 -j ACCEPT
$ sudo iptables -I FORWARD -d 192.168.126.0/24 -o virbr0 -j ACCEPT
$ sudo iptables -t nat -A POSTROUTING -s 192.168.126.0/24 ! -d 192.168.126.0/24 -j MASQUERADE

install iptables-persistent then create /etc/iptables/rules.v4

*filter
-A FORWARD -d 192.168.126.0/24 -o virbr0 -j ACCEPT
-A FORWARD -s 192.168.126.0/24 -i virbr0 -j ACCEPT

*nat
-A POSTROUTING -s 192.168.126.0/24 ! -d 192.168.126.0/24 -j MASQUERADE

Install dnsmasq to provide DHCP serverâ€‹

From: https://bobcares.com/blog/dnsmasq-dhcp-server-in-proxmox/

$ sudo apt install dnsmasq

Configure it:

domain-needed
bogus-priv
interface=vmbr0
dhcp-range=192.168.126.2,192.168.126.254,12h

Then restart the dnsmasq service.

Use systemd-resolved to provide DHCP serverâ€‹

https://www.freedesktop.org/software/systemd/man/systemd.network.html

Create a file /etc/systemd/network/proxmox.network:

[Match]
Name=vmbr*
Driver=bridge

[Network]
Address=192.168.126.1/24
DHCPServer=yes
IPMasquerade=yes

[DHCPServer]
DNS=8.8.8.8

This will enable IP forwarding etc.

Put ISO files in /var/lib/vz/template/iso
