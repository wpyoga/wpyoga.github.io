Installing OpenVZ 6

January 17, 2022

OpenVZ 6 is an obsolete piece of technology. However, I have a small project whereby I upgrade a Ubuntu 16.04 installation to Ubuntu 18.04 and then 20.04 and eventually 22.04, completely disregarding any potential issues and problems that may happen :)

Also, I may be able to research a migration strategy for OpenVZ 6 to KVM. One possible way is to install OpenVZ 6 inside a KVM virtual machine, and then migrating each container as a single container inside the new host.

The basic process is to install CentOS 6 as the base OS, then install OpenVZ 6 on top of it. After installation, we have to reboot into the newly-installed OpenVZ kernel. This is a modified kernel based off of RHEL 6 / CentOS 6 that provides the containerization features.

As a bonus, I'm installing all of this inside a KVM virtual machine.

download centos 6: https://vault.centos.org/6.10/isos/x86_64/ http://ftp.uem.br/linux/CentOS/6.10/isos/x86_64/ https://mirror.nsc.liu.se/centos-store/6.10/isos/x86_64/ http://ftp.iij.ad.jp/pub/linux/centos-vault/centos/6.10/isos/x86_64/ https://img.cs.montana.edu/linux/centos/6.10/isos/x86_64/

install centos 6 enable dhcp: https://www.serverlab.ca/tutorials/linux/administration-linux/configure-centos-6-network-settings/ (just change ONBOOT=yes)

use vault repo (since centos 6 is already EOL) https://www.getpagespeed.com/server-setup/how-to-fix-yum-after-centos-6-went-eol edit /etc/yum.repos.d/CentOS-Base.repo remove all mirrorlist= lines uncomment all baseurl= lines, change the domain to vault.centos.org

$ sudo sed -i /etc/yum.repos.d/CentOS-Base.repo -e '/^mirrorlist=/d' -e 's,^#baseurl=http://mirror.centos.org/,baseurl=https://vault.centos.org/,'

yum update

make a new user

# useradd william -G adm

change hostname

$ sudo hostname cos6
$ vi /etc/sysconfig/network
HOSTNAME=cos6

enable sudo $ visudo william add the line william ALL=(ALL) NOPASSWD:ALL

enable ssh $ mkdir /home/william/.ssh $ vi /home/william/.ssh/authorized_keys $ chmod 700 /home/william/.ssh $ chown -R william: /home/william $ ls -la /home/william/.ssh

install & enable acpid, so that the VM can be shut down gracefully by libvirt / proxmox

$ sudo yum install acpid
$ sudo service acpid start
$ sudo service acpid status

Install OpenVZ 6â€‹

then follow the instructions here: https://wiki.openvz.org/Quick_Installation_CentOS_6

$ sudo curl http://download.openvz.org/openvz.repo -o/etc/yum.repos.d/openvz.repo $ sudo rpm --import http://download.openvz.org/RPM-GPG-Key-OpenVZ $ yum install vzkernel vzctl vzquota

disable disk quota system

/etc/vz/vz.conf DISK_QUOTA=no

you can install ploop, but it is not recommended

/etc/vz/vz.conf VE_LAYOUT=simfs

make sure vzkernel is the default boot option $ sudo vi /boot/grub/menu.lst

enable ip forwarding etc $ sudo vi /etc/sysctl.conf net.ipv4.ip_forward = 1 net.ipv4.conf.default.proxy_arp = 0 net.ipv4.conf.all.rp_filter = 1 #kernel.sysrq = 1 net.ipv4.conf.default.send_redirects = 1 net.ipv4.conf.all.send_redirects = 0 net.ipv4.icmp_echo_ignore_broadcasts=1 net.ipv4.conf.default.forwarding=1

If using vethâ€‹

for VETH: create a bridge & configure vznet script $ sudo vi /etc/sysconfig/network-scripts/ifcfg-vmbr0 DEVICE="vmbr0" BOOTPROTO="static" IPV6INIT="no" ONBOOT="yes" TYPE="Bridge" DELAY=0 IPADDR=192.168.124.99 NETMASK=255.255.255.0 GATEWAY=192.168.124.1 $ sudo vi /etc/sysconfig/network-scripts/ifcfg-eth0 DEVICE=eth0 TYPE=Ethernet ONBOOT=yes IPV6INIT=no BRIDGE=vmbr0 $ sudo vi /etc/vz/vznet.conf EXTERNAL_SCRIPT=/usr/sbin/vznetaddbr

Create a containerâ€‹

create container $ sudo vzctl create 101 --ostemplate ubuntu-16.04-x86_64 --layout simfs --hostname server101 --ipadd 192.168.124.101 --diskspace 8G $ sudo vzctl set 101 --save --onboot yes --nameserver 8.8.8.8 --cpus 1 --ram 1G

use --name server101 ???

note: if using ploop, sometimes container can run out of disk space, especially when there is lots of writing and erasing. to solve this: ploop discard -d /dev/ploopXXXXX / or / vzctl compact 101 it's much simpler to just use simfs though

configure venet: https://wiki.openvz.org/Virtual_network_device

$ sudo vzctl start 101

ubuntu 16.04 template

ubuntu useradd needs extra options

$ sudo vzctl exec 101 useradd -m -s /bin/bash william $ sudo vzctl exec 101 usermod -a -G systemd-journal william $ sudo vzctl exec 101 mkdir /home/william/.ssh $ sudo vi /vz/root/101/home/william/.ssh/authorized_keys $ sudo vzctl exec 101 chmod 700 /home/william/.ssh $ sudo vzctl exec 101 chown -R william: /home/william $ sudo vzctl exec 101 ls -la /home/william/.ssh

$ sudo vi /vz/root/101/etc/sudoers william ALL=(ALL:ALL) NOPASSWD: ALL

login to the node, then $ sudo dpkg-reconfigure locales select en_US.UTF-8 to generate and select en_US.UTF-8 as the system default locale

to list all containers: vzlist --all

Configure networkingâ€‹

on the openvz host $ sudo iptables -D FORWARD -j REJECT --reject-with icmp-host-prohibited

/etc/sysconfig/iptables remove the line:

-A FORWARD -j REJECT --reject-with icmp-host-prohibited

on the physical host

$ sudo ip route add 192.168.124.0/24 via 192.168.122.223

or:

$ virsh net-edit default && virsh net-destroy default && virsh net-start default && sudo systemctl restart libvirtd

<network>
  ...
  <route family='ipv4' address='192.168.124.0' prefix='24' gateway='192.168.122.223'/>
</network>

on the physical host: copy libvirt rules

the last rule is actually the most important one

sudo iptables -D LIBVIRT_FWI -o virbr0 -j REJECT --reject-with icmp-port-unreachable sudo iptables -I FORWARD -s 192.168.124.0/24 -i virbr0 -j ACCEPT sudo iptables -I FORWARD -d 192.168.124.0/24 -o virbr0 -j ACCEPT sudo iptables -t nat -A POSTROUTING -s 192.168.124.0/24 ! -d 192.168.124.0/24 -j MASQUERADE

install iptables-persistent then create /etc/iptables/rules.v4

*filter
-A FORWARD -d 192.168.124.0/24 -o virbr0 -j ACCEPT
-A FORWARD -s 192.168.124.0/24 -i virbr0 -j ACCEPT
COMMIT

*nat
-A POSTROUTING -s 192.168.124.0/24 ! -d 192.168.124.0/24 -j MASQUERADE
COMMIT

don't save the libvirt and wireguard rules to that file, those rules will be recreated when the corresponding services are started
